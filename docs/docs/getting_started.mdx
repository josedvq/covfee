title: Getting Started
---

This guide customizes a cloned `covfee_project` starter repository to create a simple Covfee study with:

1. a demographics questionnaire
2. an instructions screen loaded from `www/instructions.md`
3. a continuous annotation task on a video
4. a final feedback questionnaire

## 1. Clone the starter project

To create a new Covfee project, start from the external `covfee_project` repository instead of creating a project folder from scratch:

```bash
git clone <your-covfee_project-repo-url> myproject
cd myproject
```

Replace `<your-covfee_project-repo-url>` with the actual Git URL of the external `covfee_project` repository. Clone that starter repository, modify the files in your clone, test the project locally, and then build the Docker deployment from that same project directory.

The cloned repository should have a layout similar to:

```text
myproject
├── app.py
├── docker/
└── www
    └── instructions.md
```

Covfee expects the study entrypoint to be `app.py`. Project assets that should be publicly available belong in `www/`.

## 2. Update the study file

Edit `app.py` in your cloned project and replace it with:

```python
from covfee import HIT, CovfeeApp, Project, tasks


task_demographics = tasks.QuestionnaireTaskSpec(
    name="Demographics",
    form={
        "fields": [
            {
                "name": "age",
                "label": "Your age:",
                "input": {"inputType": "InputNumber"},
            },
            {
                "name": "sex",
                "label": "Your sex:",
                "input": {
                    "inputType": "Select",
                    "options": [
                        {"label": "Male", "value": "m"},
                        {"label": "Female", "value": "f"},
                    ],
                },
            },
            {
                "name": "nationality",
                "label": "Your nationality:",
                "input": {"inputType": "Input"},
            },
        ]
    },
)

task_instructions = tasks.InstructionsTaskSpec(
    name="Instructions",
    content={"type": "link", "url": "$$www$$/instructions.md"},
)

task_annotation = tasks.ContinuousAnnotationTaskSpec(
    name="Arousal annotation",
    annotations=[{"name": "Arousal", "interface": "RankTrace"}],
    media={
        "type": "video",
        "url": "https://cdn.jsdelivr.net/gh/josedvq/covfee@master/samples/keypoints/www/person_reading.mp4",
    },
    userCanAdd=False,
)

task_feedback = tasks.QuestionnaireTaskSpec(
    name="Feedback",
    form={
        "fields": [
            {
                "name": "rating",
                "label": "How would you rate your experience in completing this experiment?",
                "required": True,
                "input": {"inputType": "Rate", "allowHalf": True},
            },
            {
                "name": "feedback",
                "label": "Do you have any comments that can help us improve the experience?",
                "input": {"inputType": "Input.TextArea"},
            },
        ]
    },
)

hit = HIT("Basic annotation")
hit.add_journey(
    nodes=[
        task_demographics,
        task_instructions,
        task_annotation,
        task_feedback,
    ]
)

projects = [Project("My Covfee Project", email="example@example.com", hits=[hit])]
app = CovfeeApp(projects)
```

Covfee looks for a module-level object named `app`.

## 3. Update the instructions file

Edit `www/instructions.md` in your cloned project and replace it with:

```md
# Instructions

You will be shown a short video. Please rate the speaker's arousal using your keyboard up and down arrows.
```

The `$$www$$` variable resolves to the standalone `www` file server configured for the project.

Optional entries in `covfee/dev/dev.env`:

```bash
PROJECT_DIR=/path/to/myproject
COVFEE_HOST=localhost
COVFEE_PORT=5001
```

:::tip
This example uses a publicly hosted sample video so the project stays lightweight. If you prefer a local file, place it in `www/` and change the media URL to something like `$$www$$/person_reading.mp4`.
:::

## 4. Test locally

For local development, install the main `covfee` framework repository and use it to run your cloned project with hot reload:

```bash
git clone https://github.com/josedvq/covfee.git
cd covfee
python3 -m venv ./env
source ./env/bin/activate
python3 -m pip install -e .
dev/dev-setup.sh

cd /path/to/myproject
# Edit covfee/dev/dev.env and set PROJECT_DIR=/path/to/myproject
covfee-dev run
```

This keeps your study code in the external `covfee_project` clone while using the main `covfee` repository only for framework development and the local runtime.

## 5. Annotate

The admin panel should be available at [http://localhost:5001/admin#](http://localhost:5001/admin#) by default. From there you can open HIT URLs, monitor progress, and download completed annotations.

## 6. Deploy with Docker

When you are happy with the local result, build and deploy the Docker stack from the same cloned project repository. The `docker/` folder in your `covfee_project` clone contains the deployment files. See [Deploying to a server](deployment).
