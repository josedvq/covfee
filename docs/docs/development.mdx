---
title: Development install
---

This guide explains how to install covfee for framework development. The development runtime now consists of four separate processes managed through a Procfile:

1. webpack dev server with hot reload
2. Flask/SocketIO backend with Python reload
3. standalone Node Redux store service
4. lightweight `www/` file server for project assets

## Setup

:::caution
Covfee runs on Linux, macOS, and Windows, but development tends to be smoother on Unix-like environments. On Windows we recommend [WSL](https://docs.microsoft.com/en-us/windows/wsl/install).
:::

1. Install [node.js](https://nodejs.org/en/download/).

2. Install covfee in editable mode:

```bash
git clone git@github.com:josedvq/covfee.git
cd covfee
python3 -m venv ./env
source ./env/bin/activate
python3 -m pip install -e .
```

3. Install the JavaScript dependencies and the Procfile runner:

```bash
dev/dev-setup.sh
```

4. If you changed task specs in `covfee/shared`, regenerate schemata:

```bash
covfee-dev schemata
```

## Running a project in dev mode

Projects now use a folder-based layout with an `app.py` entrypoint. To run one of the bundled samples:

```bash
cd ./samples/basic
# Edit covfee/dev/dev.env and set PROJECT_DIR=./samples/basic
covfee-dev run
```

That single command starts all four development processes and streams their logs together. `covfee-dev run` uses the shared `covfee/dev/dev.env` file for development settings, and it uses the current working directory as `PROJECT_DIR` unless you pass an explicit project directory like `covfee-dev run ./samples/basic`.

## Production bundles

When you need to rebuild the compiled frontend bundles used by deploy images:

```bash
covfee-dev build
```

## Schemata updates

The schemata file created by `covfee-dev schemata` is used by `covfee make` for validating project specifications. Re-run it whenever you change the shared task interfaces or switch branches with different task specs.
